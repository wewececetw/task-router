# Implement Simple

Use this for boilerplate-style implementation tasks:

- models, schemas, entities, and DTOs
- simple CRUD endpoints
- config files and scaffold tasks
- simple unit-test skeletons
- migration drafts
- documentation and docstrings

Ask oMLX for a draft only when the work is low-risk and repetitive. Codex must
review imports, project style, edge cases, and correctness before applying any
change.

Do not delegate auth, security, core business logic, concurrency, performance,
or data-loss-sensitive work.
